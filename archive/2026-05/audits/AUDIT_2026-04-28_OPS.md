# FIVUCSAS Ops/Infra Audit — 2026-04-28

Read-only audit by the Ops team. See companion docs
`AUDIT_2026-04-28_BASIC.md`, `AUDIT_2026-04-28_EDGE.md`,
`AUDIT_2026-04-28_SECURITY.md` (in flight at write time).

## P0 — fix today

1. **`bio.fivucsas.com` publicly exposed without middlewares**
   - `biometric-processor/docker-compose.prod.yml` carries
     `traefik.enable=true` for `Host(bio.fivucsas.com)` but **no
     `admin-whitelist@file` and no `rate-limit@file` middleware**.
   - Endpoint serves the FastAPI biometric processor and is
     reachable from any IP.
   - Compare to `identity-core-api/docker-compose.prod.yml` which
     correctly uses `secure-headers@file,rate-limit@file`.
   - **Fix**: attach both middlewares OR strip the public router
     and rely on identity-core-api proxying internally.
   - TODO Phase C4 has been open for this since 2026-04-24.

2. **Disk 91% full** (130G / 150G)
   - `docker system df`: 51 GB images (19 GB reclaimable), 23 GB
     build cache, 9.6 GB volumes (7.2 GB reclaimable).
   - Cron `docker builder prune` runs only Sundays at 03:00.
   - **Fix**: `docker builder prune -af` (safe, reclaims ~23 GB)
     immediately. Optionally `docker image prune -af` after
     verifying nothing in production references the dangling images.

## P1 — this week

3. **No observability stack running.**
   `/opt/projects/infra/observability/docker-compose.yml` is fully
   configured (Loki, Grafana, Promtail, Prometheus, Alertmanager).
   Zero observability containers running besides uptime-kuma.
   **Fix**: bring the compose stack up; configs already in place.

4. **No DR runbook; restore drill never executed.**
   Backups happen daily, GPG-encrypted, mirrored offsite. But the
   restore procedure is undocumented and never tested.
   **Fix**: write `/opt/projects/infra/RUNBOOK_DR.md`; do one full
   restore into a throwaway DB to validate.

5. **No PITR / WAL archiving — RPO 24 h.**
   Worst case: 24 hours of writes lost if VPS dies right after a
   backup runs.
   **Fix**: enable WAL archiving + offsite ship. Or accept the RPO
   and document it.

## P2 — this month

6. Postgres password hardcoded in `/opt/projects/infra/backup.sh`
   (world-readable file). **Fix**: move to env-var sourced from
   `/opt/projects/infra/.env` (`chmod 600`).
7. All built images use `:latest` — rebuild drift risk. **Fix**:
   tag images with build SHA.
8. Postgres `shared_buffers=384MB` on 16 GB host — undersized.
   **Fix**: bump to 4 GB.
9. No PgBouncer. App-level HikariCP only — fine for current load,
   no headroom.
10. Offsite Hostinger mirror has no long-term retention (mirror =
    3 days local). **Fix**: add weekly snapshot stream.
